# tg-channel-monitor

Юзербот для мониторинга Telegram-каналов по ключевым словам — сразу по нескольким независимым проектам. Отслеживает новые посты в реальном времени и отправляет форматированные алерты нужным получателям.

Построен на [Telethon](https://github.com/LonamiWebs/Telethon). Работает от обычного пользовательского аккаунта — токен бота не нужен.

---

## Как это работает

- Подписывается на `NewMessage` во всех настроенных каналах
- Сопоставляет текст поста с двумя группами ключей на проект: **персоны** и **компании**
- Извлекает наиболее релевантные предложения с найденными ключами
- Отправляет форматированный алерт с выделенными ключами, метаданными канала (тег, число подписчиков) и прямой ссылкой на пост
- Метаданные каналов (теги, подписчики) подтягиваются раз в неделю и кэшируются в `channel_meta.json`

### Формат алерта

```
📰 Название канала (12 345 подписчиков) #Ключ1 #Ключ2

Первые два предложения поста, если он длинный...

Предложение с **Ключом1** в контексте.

<...>

Другое предложение с **Ключом2**.

🔗 Пост (https://t.me/c/...)
```

Для коротких постов (≤ 3 предложений) выводится весь текст с выделенными ключами.

## Требования

- Python 3.10+
- Аккаунт Telegram (API-ключи с [my.telegram.org](https://my.telegram.org))

Установка зависимостей:

```bash
pip install telethon python-dotenv
```

## Запуск

**1. Получить API-ключи Telegram**

Зайти на [my.telegram.org](https://my.telegram.org) → API development tools → создать приложение.  
Получите `API_ID` (число) и `API_HASH` (строка).

**2. Узнать свой Telegram ID**

Напишите любое сообщение боту [@userinfobot](https://t.me/userinfobot) — он ответит вашим числовым ID.

**3. Создать `.env`**

Скопировать шаблон и заполнить значения:

```bash
cp .env.example .env
```

**4. Запустить**

```bash
python userbot.py
```

При первом запуске Telethon запросит номер телефона и код подтверждения — создаётся файл `user.session`. Последующие запуски стартуют без вопросов.

## Конфигурация

Вся настройка — в файле `.env`. Структура проектная: можно вести до 10 независимых проектов, у каждого свои каналы, ключи и получатель алертов.

### Глобальные параметры

| Переменная | Описание |
|---|---|
| `API_ID` | Telegram API ID с my.telegram.org |
| `API_HASH` | Telegram API hash с my.telegram.org |
| `MASTER_ID` | Ваш Telegram ID — получает команды `/status`, `/ping` |

### Параметры проекта

Каждый проект нумеруется `_1` … `_10`. Проект активируется, когда заданы оба поля `CHANNELS_N` и `OWNER_N`.

| Переменная | Описание |
|---|---|
| `CHANNELS_N` | Каналы через запятую: username или числовой ID |
| `OWNER_N` | Telegram ID получателя алертов по этому проекту |
| `PEOPLE_N` | Список имён персон через запятую |
| `COMPANIES_N` | Список названий компаний/организаций через запятую |

Каналы можно указывать как:
- `username` — например `durov`, `rbc_news`
- числовой ID — например `-1001234567890`

Хотя бы одно из полей `PEOPLE_N` или `COMPANIES_N` должно быть заполнено — иначе алерты по проекту не будут срабатывать.

### Теги каналов

Необязательные метки, которые выводятся в начале алерта. Удобно для обозначения типа канала или региона.

```
CHANNEL_TAGS=channelname:🔴 Срочно;otherchannel:[None]
```

Формат: `username:тег`, разделитель `;`. Значение `[None]` явно отключает тег для канала.

## Команды

Отправляются от аккаунта с `MASTER_ID` (в любом чате, где активен юзербот, или себе):

| Команда | Описание |
|---|---|
| `/ping` | Проверка работы — отвечает `pong` |
| `/status` | Список загруженных проектов с числом каналов и ID получателей |
| `/update_meta` | Принудительное обновление метаданных каналов (теги + подписчики) |

## Важные замечания

- Юзербот работает от вашего личного аккаунта. Мониторинг большого числа каналов с высокой активностью теоретически может привлечь внимание антиспам-систем Telegram — держите список каналов разумным.
- `user.session` содержит авторизованную сессию. Обращайтесь с ним как с паролем: не публикуйте, не передавайте.
- Метаданные (подписчики, теги) обновляются не чаще раза в неделю. После добавления новых каналов используйте `/update_meta` для принудительного обновления.
- Кэш дедупликации сообщений хранится в памяти (последние 10 000 сообщений) и сбрасывается при рестарте — это безопасно, Telethon не переотправляет старые события после переподключения.
