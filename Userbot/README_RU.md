# tg-channel-monitor

Юзербот для мониторинга Telegram-каналов по нескольким независимым проектам. Построен на [Telethon](https://github.com/LonamiWebs/Telethon). Работает от обычного пользовательского аккаунта — токен бота не нужен.

---

## Режимы работы

Бот запускается в одном из двух режимов. Режим задаётся через `BOT_MODE` в `.env`.

### `BOT_MODE=keyword` (по умолчанию)

Мониторит настроенные каналы и пересылает **только посты с совпадением ключей** из `PEOPLE_N` или `COMPANIES_N`. В алерте — короткий фрагмент с выделенными ключами.

```
📰 Название канала (12 345 подписчиков) #Ключ1 #Ключ2

Первые два предложения поста...

Предложение с **Ключом1** в контексте.

<...>

Другое предложение с **Ключом2**.

🔗 Пост (https://t.me/c/...)
```

`digest.py` в этом режиме не нужен.

### `BOT_MODE=digest`

Мониторит настроенные каналы и пересылает **каждый пост с текстом** без фильтрации по ключам. Длинные посты сжимаются: аннотация + ключевые тезисы.

```
📰 Название канала (12 345 подписчиков)

📌 Первое предложение или лид поста (аннотация).

Тезисы:
• Наиболее информативное предложение из поста.
• Второй ключевой тезис из другой части.
• ...

🔗 https://t.me/c/...
```

Посты короче или равные `DIGEST_SHORT_LIMIT` (по умолчанию 800 символов) всегда пересылаются целиком.

Требует `digest.py` в той же папке и виртуального окружения с Python 3.11 + NumPy 1.26.4 (см. [Установку](#установка)).

### Режим 3 (будущее)

Фильтрация по ключам + суммаризация через digest. Сейчас не реализован, архитектура позволяет добавить.

## Требования

- Python 3.11, установленный в `C:\Python311\` (см. шаг 1)
- Аккаунт Telegram (API-ключи с [my.telegram.org](https://my.telegram.org))
- Все зависимости устанавливаются автоматически через `setup_env.bat`

> **Почему Python 3.11?**
> Режим digest использует LSA-суммаризацию, которой нужен NumPy.
> NumPy 2.x требует инструкций X86_V2, которых нет на QEMU Virtual CPU 2.5+.
> NumPy 1.26.4 под Python 3.11 таких требований не имеет и работает на любом x86-64.

## Установка

### Шаг 1 — Установить Python 3.11

Скачать: [python.org/downloads/release/python-3119](https://www.python.org/downloads/release/python-3119/)
Файл: **Windows installer (64-bit)** → `python-3.11.9-amd64.exe`

При установке:
- ✅ **Add Python 3.11 to PATH**
- На экране выбора папки изменить путь на `C:\Python311\`

Проверить в CMD:
```cmd
py -3.11 --version
```
Ожидаемый результат: `Python 3.11.9`

### Шаг 2 — Скопировать файлы в папку скрипта

Положить все файлы в `C:\Python_scripts\Feedpars\`:

```
userbot.py
digest.py
.env
requirements.txt
setup_env.bat
run.bat
check_env.bat
```

### Шаг 3 — Заполнить `.env`

```cmd
copy .env.example .env
```

Заполнить как минимум: `API_ID`, `API_HASH`, `MASTER_ID`, `BOT_MODE`, `CHANNELS_1`, `OWNER_1`.

### Шаг 4 — Создать виртуальное окружение

```cmd
cd C:\Python_scripts\Feedpars
setup_env.bat
```

Скрипт сделает всё автоматически:
1. Создаст `.venv\` с Python 3.11
2. Установит все зависимости из `requirements.txt`
3. Скачает данные NLTK punkt_tab (~2 МБ, один раз)

Время: 2–5 минут.

### Шаг 5 — Проверить установку

```cmd
check_env.bat
```

В выводе должно быть:
```
[OK] NumPy 1.26.4
[OK] LSA available
```

### Шаг 6 — Запустить

```cmd
run.bat
```

При первом запуске Telethon запросит номер телефона и код подтверждения — создаётся `user.session`. Последующие запуски стартуют без вопросов.

> Всегда запускать через `run.bat`, а не `python userbot.py` напрямую — иначе бот запустится в системном Python 3.13 без нужных зависимостей.

## Конфигурация

### Глобальные параметры

| Переменная | Описание |
|---|---|
| `API_ID` | Telegram API ID |
| `API_HASH` | Telegram API hash |
| `MASTER_ID` | Ваш ID — получает команды `/status`, `/ping` |
| `BOT_MODE` | `keyword` или `digest` |

### Настройки digest (только режим digest)

| Переменная | Дефолт | Описание |
|---|---|---|
| `DIGEST_SHORT_LIMIT` | `800` | Посты ≤ этого значения целиком |
| `DIGEST_MEDIUM_LIMIT` | `1500` | Верхняя граница зоны «средний» |
| `DIGEST_LONG_LIMIT` | `3000` | Верхняя граница «длинный»; выше — лонгрид |
| `DIGEST_MEDIUM_THESES` | `2` | Тезисов для средних постов |
| `DIGEST_LONG_THESES` | `3` | Тезисов для длинных постов |
| `DIGEST_LONGREAD_THESES` | `4` | Тезисов для лонгридов |

Границы должны быть строго возрастающими. Число тезисов можно менять командами из Telegram без перезапуска.

### Параметры проекта

Каждый проект нумеруется `_1` … `_10`. Активируется при наличии `CHANNELS_N` и `OWNER_N`.

| Переменная | Описание |
|---|---|
| `CHANNELS_N` | Каналы через запятую: username или числовой ID |
| `OWNER_N` | Telegram ID получателя алертов |
| `PEOPLE_N` | Имена персон (только режим keyword) |
| `COMPANIES_N` | Названия компаний/организаций (только режим keyword) |

### Теги каналов

```
CHANNEL_TAGS=rbc_news:🔴;kommersant:📰;other:[None]
```

Необязательные метки в шапке алерта. `[None]` явно отключает тег.

## Команды

### Базовые (только MASTER_ID)

| Команда | Описание |
|---|---|
| `/ping` | Проверка работы |
| `/status` | Проекты, каналы, текущий режим |
| `/update_meta` | Принудительное обновление метаданных каналов |

### Шкала digest (MASTER_ID + все OWNER_N, только режим digest)

Настройки сохраняются в `digest_config.json` между перезапусками.

| Команда | Пример | Описание |
|---|---|---|
| `/digest_status` | | Текущая шкала и число тезисов |
| `/digest_set` | `/digest_set short 600` | Граница зоны в символах |
| `/digest_theses` | `/digest_theses longread 5` | Число тезисов для зоны |
| `/digest_reset` | | Сброс к дефолтам из `.env` |

Зоны для `/digest_set`: `short`, `medium`, `long`
Зоны для `/digest_theses`: `medium`, `long`, `longread`

## Решение проблем

**`py -3.11` не найден после установки** — закройте и откройте CMD. Если не помогает: проверьте что `C:\Python311\` и `C:\Python311\Scripts\` есть в переменной PATH.

**Ошибка NumPy при запуске** — вы запустили `python userbot.py` напрямую вместо `run.bat`. Всегда используйте `run.bat`.

**Сбросить окружение полностью:**
```cmd
rmdir /s /q .venv
setup_env.bat
```

## Важные замечания

- Бот работает от личного аккаунта. Держите список каналов разумным.
- `user.session` — авторизованная сессия. Обращайтесь как с паролем.
- В режиме digest каждый пост проходит суммаризацию: ~0,1–0,5 с на пост (LSA, только CPU).
- `digest_config.json` хранит текущие настройки шкалы. Удалите файл или выполните `/digest_reset` для возврата к дефолтам.
- Виртуальное окружение `.venv\` полностью изолировано — другие скрипты на сервере не затронуты.
