# tg-channel-monitor

Юзербот для мониторинга Telegram-каналов по нескольким независимым проектам. Построен на [Telethon](https://github.com/LonamiWebs/Telethon). Работает от обычного пользовательского аккаунта — токен бота не нужен.

---

## Режимы работы

Бот запускается в одном из двух независимых режимов. Режим задаётся через `BOT_MODE` в `.env`.

### `BOT_MODE=keyword` (по умолчанию)

Мониторит настроенные каналы и пересылает **только посты с совпадением ключей** из `PEOPLE_N` или `COMPANIES_N`. В алерте — короткий фрагмент с выделенными ключами.

```
📰 Название канала (12 345 подписчиков) #Ключ1 #Ключ2

Первые два предложения поста...

Предложение с **Ключом1** в контексте.

<...>

Другое предложение с **Ключом2**.

🔗 Пост (https://t.me/c/...)
```

`digest.py` в этом режиме не нужен.

### `BOT_MODE=digest`

Мониторит настроенные каналы и пересылает **каждый пост с текстом** — без фильтрации по ключам. Длинные посты сжимаются: аннотация + ключевые тезисы. Ключевые слова не используются.

```
📰 Название канала (12 345 подписчиков)

📌 Первый абзац или вводное предложение (аннотация).

Тезисы:
1. Наиболее информативное предложение из поста.
2. Второй ключевой тезис из другой части.
3. ...

🔗 https://t.me/c/...
```

Посты короче или равные `DIGEST_SHORT_LIMIT` (по умолчанию 800 символов) всегда пересылаются целиком.

Требует `digest.py` в той же папке и установленного `sumy`.

### Режим 3 (будущее)

Фильтрация по ключам + суммаризация через digest. Сейчас не реализован, но архитектура позволяет добавить.

## Требования

- Python 3.10+
- Аккаунт Telegram (API-ключи с [my.telegram.org](https://my.telegram.org))

**Режим keyword:**
```bash
pip install telethon python-dotenv
```

**Режим digest (дополнительно):**
```bash
pip install sumy
python -c "import nltk; nltk.download('punkt_tab')"
```

`punkt_tab` скачивается один раз (~2 МБ) и кэшируется локально.

## Запуск

**1.** Зайти на [my.telegram.org](https://my.telegram.org) → API development tools → создать приложение. Получить `API_ID` и `API_HASH`.

**2.** Узнать свой Telegram ID через [@userinfobot](https://t.me/userinfobot).

**3.** Скопировать и заполнить конфиг:
```bash
cp .env.example .env
```

**4.** Установить `BOT_MODE=keyword` или `BOT_MODE=digest` в `.env`.

**5.** Запустить:
```bash
python userbot.py
```

При первом запуске Telethon запросит номер телефона и код подтверждения — создаётся `user.session`. Последующие запуски стартуют без вопросов.

## Конфигурация

### Глобальные параметры

| Переменная | Описание |
|---|---|
| `API_ID` | Telegram API ID |
| `API_HASH` | Telegram API hash |
| `MASTER_ID` | Ваш ID — получает команды `/status`, `/ping` |
| `BOT_MODE` | `keyword` или `digest` |

### Настройки digest (только для режима digest)

| Переменная | Дефолт | Описание |
|---|---|---|
| `DIGEST_SHORT_LIMIT` | `800` | Посты ≤ этого значения целиком |
| `DIGEST_MEDIUM_LIMIT` | `1500` | Верхняя граница зоны «средний» |
| `DIGEST_LONG_LIMIT` | `3000` | Верхняя граница «длинный»; выше — лонгрид |
| `DIGEST_MEDIUM_THESES` | `2` | Тезисов для средних постов |
| `DIGEST_LONG_THESES` | `3` | Тезисов для длинных постов |
| `DIGEST_LONGREAD_THESES` | `4` | Тезисов для лонгридов |

Границы должны быть строго возрастающими. Число тезисов можно менять командами из Telegram.

### Параметры проекта

Каждый проект нумеруется `_1` … `_10`. Активируется при наличии `CHANNELS_N` и `OWNER_N`.

| Переменная | Описание |
|---|---|
| `CHANNELS_N` | Каналы через запятую: username или числовой ID |
| `OWNER_N` | Telegram ID получателя алертов |
| `PEOPLE_N` | Имена персон (только режим keyword) |
| `COMPANIES_N` | Названия компаний/организаций (только режим keyword) |

### Теги каналов

```
CHANNEL_TAGS=rbc_news:🔴;kommersant:📰;other:[None]
```

Необязательные метки в шапке алерта. `[None]` явно отключает тег.

## Команды

### Базовые (только MASTER_ID)

| Команда | Описание |
|---|---|
| `/ping` | Проверка работы |
| `/status` | Проекты, каналы, текущий режим |
| `/update_meta` | Принудительное обновление метаданных |

### Шкала digest (MASTER_ID + все OWNER_N, только режим digest)

Настройки сохраняются в `digest_config.json` между перезапусками.

| Команда | Пример | Описание |
|---|---|---|
| `/digest_status` | | Текущая шкала и число тезисов |
| `/digest_set` | `/digest_set short 600` | Граница зоны в символах |
| `/digest_theses` | `/digest_theses longread 5` | Число тезисов для зоны |
| `/digest_reset` | | Сброс к дефолтам из `.env` |

Зоны для `/digest_set`: `short`, `medium`, `long`
Зоны для `/digest_theses`: `medium`, `long`, `longread`

## Важные замечания

- Бот работает от личного аккаунта. Держите список каналов разумным.
- `user.session` — авторизованная сессия. Обращайтесь как с паролем.
- В режиме digest каждый пост проходит через суммаризацию. На слабом железе это ~0,1–0,5 с на пост (LSA, только CPU, GPU не нужен).
- `digest_config.json` хранит текущие настройки шкалы. Удалите файл или выполните `/digest_reset` для возврата к дефолтам из `.env`.
