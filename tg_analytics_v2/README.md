# TG Analytics — руководство пользователя

Скрипт собирает статистику постов в Telegram-каналах и раз в неделю
и раз в месяц присылает вам Excel-отчёт прямо в Telegram.

---

## Что нужно сделать один раз перед запуском

### 1. Установить Python

Скачайте Python 3.11 с официального сайта: https://www.python.org/downloads/

При установке обязательно поставьте галочку **«Add Python to PATH»**.

Проверьте что установился — откройте CMD и напишите:
```
python --version
```
Должно написать что-то вроде `Python 3.11.9`.

---

### 2. Скачать файлы скрипта

Распакуйте архив в любую папку, например `C:\tg_analytics\`

В папке должны быть эти файлы:
```
main.py
snapshot.py
report.py
auth.py
.env.example
requirements.txt
README.md
```

---

### 3. Установить библиотеки

Откройте CMD, перейдите в папку со скриптом и выполните команду:

```
cd C:\tg_analytics
pip install -r requirements.txt
```

Это нужно сделать только один раз. Команда скачает всё необходимое.

---

### 4. Получить Telegram API ключи

Это нужно чтобы скрипт мог читать ваши каналы.

1. Откройте https://my.telegram.org в браузере
2. Войдите под своим номером телефона
3. Нажмите **API development tools**
4. Заполните форму (название и описание — любые, например «Analytics»)
5. Нажмите **Create application**
6. Скопируйте **App api_id** и **App api_hash** — они понадобятся на следующем шаге

---

### 5. Заполнить файл настроек

В папке со скриптом найдите файл `.env.example`.
Скопируйте его и переименуйте копию в `.env` (без слова «example»).

Откройте `.env` в Блокноте и заполните:

```
API_ID=        ← вставьте App api_id с my.telegram.org
API_HASH=      ← вставьте App api_hash с my.telegram.org

CHANNELS=@channel_one,@channel_two   ← ваши каналы через запятую

REPORT_RECIPIENT_ID=   ← ваш Telegram ID (узнаете при первом запуске)
DEBUG_RECIPIENT_ID=    ← Telegram ID резервного аккаунта для тестовых отчётов
```

**Telegram ID** — это число, не путайте с @username.
Узнать свой ID можно написав боту @userinfobot в Telegram.
Скрипт тоже покажет ваш ID при первом запуске.

**Режим отладки** — пока тестируете, поставьте:
```
DEBUG=true
```
В этом режиме скрипт каждый день в 12:00 будет присылать суточный отчёт
на `DEBUG_RECIPIENT_ID`. Так вы сразу увидите что всё работает.
Когда убедитесь — поменяйте на `DEBUG=false`.

---

## Запуск

Откройте CMD, перейдите в папку и запустите:

```
cd C:\tg_analytics
python main.py
```

При первом запуске скрипт попросит ввести номер телефона и код из Telegram —
это разовая авторизация, потом она больше не потребуется.

После авторизации скрипт начнёт работать сам:
- **Каждый час** — проверяет новые посты, фиксирует время публикации
- Ровно через **24 часа** после каждого поста — снимает финальную статистику
- **Каждый понедельник в 10:00** — отправляет недельный отчёт
- **3-го числа каждого месяца в 10:00** — отправляет месячный отчёт
- **Каждый день в 12:00** (только при `DEBUG=true`) — отправляет суточный отчёт

Окно CMD можно свернуть — скрипт продолжит работать.
Чтобы остановить — нажмите `Ctrl+C` в окне CMD.

## Что приходит в Telegram

**Суточный отчёт** (только в режиме отладки):
- Лист «Посты» — каждый пост за вчера, все метрики
- Лист «По дням» — сводка дней с графиками

**Недельный отчёт**:
- Лист «Посты» — каждый пост за прошлую неделю
- Лист «По дням» — разбивка по дням
- Лист «По неделям» — итог недели

**Месячный отчёт** (два файла):
- Файл на каждый канал: посты, по дням, по неделям
- Сводный файл: все каналы на одном листе для сравнения

Во всех файлах есть вкладка **«Пояснения к показателям»** — там объяснено
что означает каждая цифра и как она считается.

---

## Показатели в отчётах

| Показатель | Что означает |
|---|---|
| Охват (Views) | Сколько раз просмотрели пост |
| Реакции | Сумма всех лайков, 🔥, ❤️ и других реакций |
| Комментарии | Количество комментариев |
| Пересылки | Сколько раз переслали пост |
| Голоса | Сумма голосов в опросе (если был) |
| Число действий | Сумма всего вышеперечисленного |
| ERR % | Какой % просмотревших что-то сделали. Норма: 3–8% |
| ER % | Вовлечённость от числа подписчиков |
| VRpost % | Какой % подписчиков увидел пост |
| Viral Factor % | Какой % поделился постом |
| CQI | Взвешенная оценка качества поста (комменты ценятся больше лайков) |

## Если что-то пошло не так

**Скрипт не запускается** — проверьте что Python установлен и библиотеки
установлены через `pip install -r requirements.txt`.

**Ошибка авторизации** — проверьте `API_ID` и `API_HASH` в файле `.env`.
Убедитесь что скопировали их без лишних пробелов.

**Отчёты не приходят** — проверьте `REPORT_RECIPIENT_ID` в `.env`.
Это должно быть число, а не @username. Также проверьте файл `logs/main.log` —
там записано что происходит.

**Скрипт упал и закрылся** — откройте `logs/main.log` и посмотрите
последние строки с ошибкой.
