# TG Analytics

Инструмент для автоматического сбора и анализа статистики постов и сторис в Telegram-каналах. Собирает данные через 24 часа после публикации каждого поста, отслеживает динамику подписчиков и формирует подробные Excel-отчёты с доставкой прямо в Telegram.

---

## Возможности

- Автоматический сбор статистики по постам через ровно 24 часа после публикации
- Сбор статистики по сторис (просмотры, реакции для своих каналов)
- Ежедневный учёт динамики подписчиков (+сегодня / +неделя / +месяц)
- Суточные, недельные и месячные Excel-отчёты с графиками
- Исторический сбор данных за периоды до запуска скрипта (ретро-режим)
- Кэширование исторических данных — каждый месяц читается из Telegram один раз
- Отправка отчётов прямо в Telegram, нескольким получателям
- Управление через команды в Telegram (`/report`, `/backfill`)
- Поддержка нескольких каналов одновременно
- Режим отладки с ежедневными тестовыми отчётами на резервный аккаунт
- Один файл для запуска — никаких внешних планировщиков, всё работает изнутри `main.py`
- Дедупликация альбомов (несколько фото в одном посте считаются как один пост)

## Метрики

| Метрика | Формула | Описание |
|---|---|---|
| Охват (Views) | — | Просмотры поста |
| Реакции | — | Сумма всех эмодзи-реакций |
| Комментарии | — | Число комментариев |
| Пересылки | — | Число репостов |
| Голоса | — | Сумма голосов в опросе |
| Число действий | React + Comments + Forwards + Votes | Совокупная активность (без охвата) |
| ERR % | Actions / Views × 100% | Engagement Rate by Reach — вовлечённость от охвата |
| ER % | Actions / Подписчики × 100% | Engagement Rate — вовлечённость от базы подписчиков |
| VRpost % | Views / Подписчики × 100% | View Rate — какой % подписчиков увидел пост |
| Viral Factor % | Forwards / Views × 100% | Доля поделившихся постом |
| Reply Rate % | Comments / Views × 100% | Насколько пост провоцирует диалог |
| Reach Multiplier | Views / Подписчики | Превышение охвата над базой подписчиков |
| CQI | (React×1 + Votes×2 + Fwd×4 + Comments×5) / Views × 100 | Content Quality Index — взвешенный индекс качества |

Веса CQI настраиваются в `.env`.

### Модули — что делает каждый

**`main.py`**
Единая точка входа. При первом запуске проводит авторизацию (QR-код или номер телефона). Дальше работает в бесконечном цикле: каждую минуту проверяет расписание и каждый час запускает сборщики. Слушает команды от пользователя в Telegram (`/report`, `/backfill`) и ведёт диалоги (выбор месяца, недели, кэш/пересчёт).

**`snapshot.py`**
Почасовой сборщик. Сканирует каналы на новые посты, регистрирует их с меткой `deadline = время_публикации + 24ч`. Снимает финальный срез статистики только у постов, для которых этот срок истёк. Дедуплицирует альбомы — из нескольких фото одного поста берёт только первое (с привязанными реакциями). Раз в день записывает текущее число подписчиков в историю.

**`historical.py`**
Модуль работы с историческими данными. Если для запрошенного периода (неделя, месяц) нет 24ч-срезов в реестре — читает историю канала напрямую из Telegram и кэширует результат по месяцам. Повторные запросы за тот же период не обращаются к Telegram повторно.

**`stories.py`**
Аналог `snapshot.py`, но для сторис. Регистрирует активные сторис, через 24 часа (срок жизни сторис) снимает финальную статистику просмотров. Реакции доступны только для каналов, где аккаунт является администратором.

**`report.py`**
Строит все Excel-отчёты: суточный, недельный, месячный. Содержит общие функции для построения листов с постами (группировка по каналам), агрегатных листов "По дням"/"По неделям" с графиками, сводного листа и листа с пояснениями к показателям. Отправляет готовые файлы в Telegram.

**`auth.py`**
Отдельный скрипт для ручной первичной авторизации (если не хотите проходить её через `main.py`). Поддерживает QR-код и вход по номеру телефона.

## Хранимые данные (создаются автоматически)

```
registry/
└── <channel_username>/
    ├── registry.json              # активные и финальные 24ч-срезы постов
    ├── subscribers_history.json   # дневная история числа подписчиков
    ├── stories.json               # активные и финальные срезы сторис
    └── historical/
        └── YYYY-MM.json           # кэш исторических данных за месяц

archive/
└── <channel_username>/
    └── archive_YYYY-MM.json       # архив 24ч-срезов после генерации месячного отчёта

output/
├── daily/
│   └── daily_YYYY-MM.xlsx
├── weekly/
│   └── weekly_YYYY-MM.xlsx
└── monthly/
    └── monthly_YYYY-MM.xlsx       # итоговые отчёты, отправляемые в Telegram

logs/
├── main.log
├── snapshot.log
└── report.log

tg_analytics.session                # файл авторизации Telethon (создаётся при первом входе)
```

| Файл/папка | Что хранит | Когда создаётся |
|---|---|---|
| `registry.json` | Посты в работе и с финальным 24ч-срезом | При каждом запуске сборщика |
| `subscribers_history.json` | Число подписчиков по датам | Раз в день при сборе |
| `stories.json` | Сторис в работе и с финальным срезом | При каждом запуске сборщика |
| `historical/YYYY-MM.json` | Полный месяц постов, прочитанный из истории Telegram | При запросе периода без 24ч-данных |
| `archive_YYYY-MM.json` | Снятые с реестра данные после месячного отчёта | После генерации месячного отчёта |
| `*.xlsx` в `output/` | Готовые отчёты | При каждой генерации отчёта |
| `tg_analytics.session` | Сессия авторизации Telegram | При первом запуске |

## Установка

```bash
git clone https://github.com/your-username/tg-analytics.git
cd tg-analytics
pip install -r requirements.txt
cp .env.example .env
```

### Заполнение `.env`

```env
# Telegram API — получить на https://my.telegram.org
API_ID=12345678
API_HASH=abcdef1234567890abcdef1234567890
SESSION_NAME=tg_analytics

# Каналы для анализа — через запятую
CHANNELS=@channel_one,@channel_two

# Получатели отчётов — Telegram ID, можно несколько через запятую
REPORT_RECIPIENT_ID=123456789,987654321

# Режим отладки
DEBUG=false
DEBUG_RECIPIENT_ID=123456789
DAILY_REPORT_TIME=12:00

# Часовой пояс
TIMEZONE=Europe/Moscow

# Веса CQI
CQI_W_REACT=1
CQI_W_VOTE=2
CQI_W_FORWARD=4
CQI_W_COMMENT=5
```

Узнать свой Telegram ID можно у бота [@userinfobot](https://t.me/userinfobot).

## Запуск

```bash
python main.py
```

При первом запуске потребуется авторизация — выберите способ:
- **QR-код** — отсканировать в Telegram → Настройки → Устройства → Подключить устройство
- **Номер телефона** — ввести номер и код из Telegram

После авторизации скрипт работает автономно. Окно можно свернуть, для остановки — `Ctrl+C`.

### Windows — автозапуск при старте системы

Создайте `run.bat`:
```bat
@echo off
cd /d C:\path\to\tg_analytics
python main.py
pause
```
Скопируйте в папку автозагрузки: `Win+R` → `shell:startup`.

## Расписание (встроено в `main.py`, без внешних cron/Task Scheduler)

| Действие | Когда |
|---|---|
| Сбор постов и сторис | каждый час |
| Запись подписчиков | раз в день, при первом сборе после полуночи |
| Суточный отчёт | ежедневно в `DAILY_REPORT_TIME` (только если `DEBUG=true`), на `DEBUG_RECIPIENT_ID` |
| Недельный отчёт | каждый понедельник в 10:00, на `REPORT_RECIPIENT_ID` |
| Месячный отчёт | 3-го числа каждого месяца в 10:00, на `REPORT_RECIPIENT_ID` |

## Команды в Telegram

Пишите аккаунту юзербота (или с того аккаунта, что указан в `REPORT_RECIPIENT_ID`/`DEBUG_RECIPIENT_ID`):

```
/report daily      — суточный отчёт прямо сейчас
/report weekly      — недельный отчёт (предложит выбрать неделю)
/report monthly      — месячный отчёт (спросит за какой месяц, предложит кэш или пересчёт)
/backfill           — ретро-сбор исторических данных за выбранный период
```

## Структура Excel-отчётов

### Месячный (`monthly_YYYY-MM.xlsx`)
1. **Сводка** — строка = канал: подписчики, прирост, посты, сторис, охваты, все метрики
2. **Посты 24ч** — все посты всех каналов подряд, статистика через 24 ч после публикации
3. **Посты-месяц** — то же самое, но накопленная статистика на дату сбора
4. **По дням** — канал за каналом, 31 строка + итоги, с графиками
5. **По неделям** — канал за каналом, 4-5 строк + итоги, с графиками
6. **Пояснения к показателям** — формулы и описания всех метрик

### Недельный / суточный
1. **Посты** — все каналы подряд
2. **По дням** — канал за каналом
3. **По неделям** (только в недельном) — канал за каналом
4. **Пояснения к показателям**

## Частые вопросы

**Можно ли добавить канал в середине месяца?**
Да — добавьте в `CHANNELS` в `.env`. Сбор начнётся со следующего часового тика.

**Как изменить веса CQI?**
Поменяйте `CQI_W_*` в `.env`. Пересчёт применится при следующей генерации отчёта.

**Что если пост — альбом из нескольких фото?**
Скрипт автоматически берёт только первое фото альбома (к нему привязаны реакции и комментарии всего альбома), остальные пропускает — задвоения не будет.

**Где взять токен для сторис с реакциями?**
Реакции на сторис доступны только если ваш аккаунт — администратор канала с правом редактирования сторис. Для чужих каналов будут собираться только просмотры.

**Что делать если отчёт за период уже был сформирован?**
При повторном запросе `/report monthly` скрипт предложит отправить готовый файл или пересчитать заново (со сбросом кэша).
